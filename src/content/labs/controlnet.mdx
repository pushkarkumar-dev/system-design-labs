## Why this lab exists

Every controlled image generation pipeline you interact with — interior design tools that preserve room layout, character generators that hold body pose, sketch-to-photo systems — uses ControlNet under the hood. Stable Diffusion without ControlNet generates images that are "vaguely like" what you describe. ControlNet makes it generate "exactly this spatial structure."

You have read that ControlNet "adds a parallel branch to the UNet." You have seen diagrams with arrows and boxes labeled "zero-conv." But until you have implemented a 5-step Canny edge detector from scratch, watched the zero-initialized final convolution guarantee that the control signal contributes exactly zero at training step 0, built a DDIM reverse loop that interleaves noise prediction with spatial conditioning, and stacked three preprocessors additively — you do not actually know ControlNet.

This lab builds the complete ControlNet architecture in three stages: image preprocessors in ~200 lines, conditioning injection with zero-initialization in ~280 lines, and a multi-control DDIM generation pipeline in ~300 lines. The reference implementation uses a stub UNet (3 Conv2d layers) rather than a real Stable Diffusion checkpoint, so the entire pipeline runs on any laptop in seconds without a GPU or multi-GB download.

## What we're building

A ControlNet integration in Python, implemented across three stages. Each stage adds one architectural layer and is immediately runnable.

**We build:**
- `CannyPreprocessor`: pure NumPy 5-step edge detector (blur, Sobel, NMS, threshold, hysteresis)
- `DepthPreprocessor`: gradient proxy that returns a top-dark, bottom-bright depth map without downloading MiDaS
- `PosePreprocessor`: synthetic skeleton overlay drawn with PIL without downloading OpenPose
- `ControlNetLayer`: single zero-initialized conditioning layer (`output = sample + zero_conv(control) * scale`)
- `ControlNetStack`: list of `(ControlNetLayer, float)` pairs with additive conditioning
- `ConditioningInjector`: wraps the stub UNet, encodes the control hint, injects at each encoder stage
- `ControlledDiffusionPipeline`: DDIM reverse loop with ControlNet conditioning at each step
- `MultiControlPipeline`: stacks multiple preprocessors, combines conditioning signals additively
- FastAPI server: `POST /generate`, `GET /modes`, `GET /health`
- Spring Boot Java integration: `ControlNetClient` calling the server with base64-encoded control images

**We explicitly skip (and why):**
- Real Stable Diffusion UNet checkpoint: requires 860M parameters and a multi-GB download; covered in "What the Toy Misses"
- Classifier-free guidance: doubles the forward pass; described in "What the Toy Misses"
- VAE encoding/decoding: real SD operates in latent space; our stub operates directly in pixel space
- SDXL compatibility: different UNet architecture; SD 1.5 structure is the foundational case

## Mental model

ControlNet solves the problem of spatial control in diffusion models. A text prompt says "a building." ControlNet says "a building with exactly these edges" — and the edges come from a photograph of the real building.

The architecture adds a trainable copy of the UNet encoder that processes the control image in parallel with the noisy latent:

```
Control image (edges/depth/pose)
        |
  [ControlNet encoder copy]
        |
  zero_conv(control)    <-- zero at training step 0
        |
        + ---------> UNet encoder output at each stage
                          |
                     [UNet decoder]
                          |
                    Predicted noise
```

The zero-initialization of `zero_conv` is the critical insight. At training step 0:

```
zero_conv.weight = 0  (explicitly initialized)
zero_conv(any_tensor) = 0

output = sample + 0 * scale = sample
```

The base UNet is mathematically unaffected at step 0. Fine-tuning begins from the exact pretrained UNet behavior. Without zero-init, the control signal immediately distorts UNet outputs — it takes roughly 3x more training steps to recover from the noise introduced by a random initialization.

The conditioning scale controls the tradeoff between prompt and spatial control:

```
scale = 0.0  ->  zero conditioning, behaves like unconditioned diffusion
scale = 0.5  ->  soft control, allows style variation within structure
scale = 1.0  ->  full control, edges/depth/pose are followed strictly
scale = 2.0  ->  over-conditioning, may produce artifacts
```

DDIM (Denoising Diffusion Implicit Models) replaces the DDPM stochastic sampler with a deterministic update rule. The update at each step is:

```
x0_pred = (x_t - sqrt(1 - alpha_t) * noise_pred) / sqrt(alpha_t)
x_{t-1} = sqrt(alpha_{t-1}) * x0_pred + sqrt(1 - alpha_{t-1}) * noise_pred
```

This is deterministic given the same seed: same noise `->` same output. It also allows skipping steps — 20 DDIM steps produce quality comparable to 1000 DDPM steps.

The Canny edge detector is the canonical ControlNet preprocessor — it converts any photo into a line drawing that guides generation. Implementing it from scratch reveals why it works as a conditioning signal: it extracts the spatial skeleton of an image (edges, contours, object boundaries) while discarding texture and color.

**Why a 5-step pipeline?**

Naive edge detection (`abs(diff(pixels))`) picks up noise as edges. The Canny pipeline solves this with four additional steps:

1. Gaussian blur suppresses noise before gradient computation — a blurred pixel represents its neighbourhood, not single-pixel noise.
2. Sobel gradients compute the edge strength (magnitude) and direction (angle) at each pixel.
3. Non-maximum suppression thins wide gradient ridges to single-pixel edges by comparing each pixel to its neighbours in the gradient direction.
4. Double threshold classifies pixels as strong (definitely edge), weak (possibly edge), or suppressed.
5. Hysteresis connects weak pixels to strong ones, preserving continuous edges and discarding isolated noise pixels.

**The Sobel step:**

```python
Kx = [[-1, 0, 1], [-2, 0, 2], [-1, 0, 1]]  # horizontal gradient
Ky = [[-1, -2, -1], [0, 0, 0], [1, 2, 1]]  # vertical gradient

Gx = convolve(img, Kx)
Gy = convolve(img, Ky)
magnitude = sqrt(Gx^2 + Gy^2)
angle = arctan2(Gy, Gx) % 180  # 0..180 degrees (direction of edge)
```

The angle is quantized to 4 directions (0, 45, 90, 135 degrees) for NMS. A horizontal edge has angle ~0 degrees, so NMS compares the pixel to its left and right neighbours. A diagonal edge at 45 degrees compares to the top-right and bottom-left neighbours.

**The hysteresis step:**

```python
# After double_threshold: strong=255, weak=128, suppressed=0
# Iteratively promote weak pixels adjacent to strong ones:
while changed:
    for each pixel where value == 128:
        if any 8-connected neighbour == 255:
            set pixel to 255
            changed = True
# Any remaining weak pixels are suppressed to 0
```

This is why Canny produces clean, continuous edge lines rather than scattered bright points — the hysteresis step connects weak edge fragments into coherent curves.

**DepthPreprocessor and PosePreprocessor are stubs** that return valid-format images without downloading multi-hundred-MB models. The depth stub returns a vertical gradient (top=0, bottom=255) which approximates the convention used by ControlNet-depth: dark pixels are near, bright pixels are far. The pose stub draws a synthetic 14-keypoint skeleton using PIL Draw. Both can be replaced by production implementations (`transformers` depth estimation, `controlnet_aux` OpenPose) by changing only the `process()` method.

The zero-initialization of `zero_conv` is the architectural insight that makes ControlNet trainable without destabilizing the base UNet. Understanding it requires understanding what happens without it.

**Without zero-init (random init):**

At training step 0, the control branch outputs random noise. The UNet encoder receives `sample + random_noise * scale`. The denoising loss is computed against this distorted prediction. Gradients flow through the distortion — the model must first "unlearn" the random initialization before it can learn the control signal. Empirically this takes 3x more gradient steps.

**With zero-init:**

```python
self.zero_conv = nn.Conv2d(channels, channels, kernel_size=1)
nn.init.zeros_(self.zero_conv.weight)  # weight = 0
nn.init.zeros_(self.zero_conv.bias)    # bias = 0

# At step 0:
# zero_conv(control) = 0 @ control + 0 = 0  (regardless of control)
# output = sample + 0 * scale = sample
```

The UNet at step 0 is mathematically identical to the pretrained base UNet. The first gradient update is against a meaningful signal (the pretrained model's prediction), not against noise.

**The scale parameter:**

```python
self.scale = nn.Parameter(torch.ones(1))

def forward(self, sample, control):
    return sample + self.zero_conv(control) * self.scale
```

`scale` is a learnable scalar initialized to 1.0. During fine-tuning, the model learns the optimal injection strength per layer. In inference, you can override it with the `conditioning_scale` argument to interpolate between unconditioned (`scale=0`) and fully conditioned (`scale=1`).

**ControlNetStack — multiple layers:**

```python
class ControlNetStack(nn.Module):
    def __init__(self, channels_list, conditioning_scales=None):
        self.layers = nn.ModuleList([ControlNetLayer(c) for c in channels_list])
        self.conditioning_scales = conditioning_scales or [1.0] * len(channels_list)

    def forward(self, sample, controls):
        out = sample
        for layer, ctrl, scale in zip(self.layers, controls, self.conditioning_scales):
            out = layer(out, ctrl * scale)
        return out
```

Each encoder level of the UNet has a corresponding ControlNetLayer. A real SD 1.5 UNet has 12 encoder stages; our stub has 3. The conditioning signals are summed, not averaged — this means two control signals at scale=1.0 add their contributions, which is why over-conditioning (scale above 1.5) can produce artifacts.

**ConditioningInjector** wraps the stub UNet and a control encoder with matching architecture. The control encoder maps the preprocessed control image to feature maps at each stage. At each stage, the ControlNetLayer adds the zero-initialized residual.

The DDIM sampler is the key to efficient controlled generation. DDPM (the original diffusion sampler) requires 1000 steps. DDIM achieves similar quality in 20 steps by using a deterministic update rule derived from the same noise schedule.

**The noise schedule:**

```python
betas = linspace(0.0001, 0.02, T)    # variance of noise added at step t
alphas = 1 - betas                    # signal retention at step t
alpha_cumprod = cumprod(alphas)       # cumulative signal retention

# At step t, the noisy image is:
# x_t = sqrt(alpha_cumprod[t]) * x0 + sqrt(1 - alpha_cumprod[t]) * noise
```

The `alpha_cumprod` formula means you can jump directly to any noise level without running the forward process step by step. `alpha_cumprod[0] ≈ 1.0` (almost no noise), `alpha_cumprod[T-1] ≈ 0.0` (almost pure noise).

**DDIM reverse step:**

```
Given x_t and noise prediction eps_pred:
  x0_pred = (x_t - sqrt(1 - a_t) * eps_pred) / sqrt(a_t)   # predicted clean image
  x_{t-1} = sqrt(a_{t-1}) * x0_pred + sqrt(1 - a_{t-1}) * eps_pred
```

This is deterministic — no random sampling. The same starting noise and same seed produce the same output every time. DDPM adds a stochastic term; DDIM removes it. This is why DDIM results are reproducible and why inversion (finding the noise that produced a given image) is possible.

**ControlNet injection at each step:**

```python
for t in reversed(range(n_steps)):
    a = schedule.alpha_cumprod[t]

    # ControlNet conditions the noise prediction at every denoising step
    noise_pred = injector.inject(x, control_tensor * scale)

    # DDIM update
    x0_pred = (x - sqrt(1 - a) * noise_pred) / sqrt(a)
    x0_pred = x0_pred.clamp(-1, 1)
    dir_xt = sqrt(1 - a_prev) * noise_pred
    x = sqrt(a_prev) * x0_pred + dir_xt
```

The control signal is applied at every step — not just once. Each denoising step pushes the image slightly toward the conditioned direction. Over 20 steps, this produces output that strongly follows the control structure.

**MultiControlPipeline — additive stacking:**

```python
combined_control = None
for ctrl_img, mode, ctrl_scale in conditions:
    preprocessor = PREPROCESSORS[mode]()
    processed = preprocessor.process(ctrl_img)
    ctrl_tensor = resize_control(processed, size=size) * ctrl_scale
    combined_control = ctrl_tensor if combined_control is None \
                       else combined_control + ctrl_tensor

# Clamp to prevent runaway signals from sum of multiple scales
combined_control = combined_control.clamp(-2.0, 2.0)
```

Each control signal contributes proportionally to its scale. A Canny signal at scale=1.0 and a depth signal at scale=0.5 means edges have twice the influence of depth on the denoising direction. The clamping prevents the combined signal from exceeding a range where the UNet produces artifacts.

## Stress tests

**Canny pipeline correctness under all-black and all-white images**: a uniform image should produce zero edges (no gradient to detect). Verify that both `all_black.png` and `all_white.png` inputs produce output images where every pixel is 0. If hysteresis is buggy, isolated noise pixels may survive.

**Zero-init property across 1000 random controls**: for each of 1000 random control tensors, verify that `zero_conv(control).abs().max() < 1e-8`. This confirms that zero-initialization holds regardless of input magnitude or shape.

**DDIM determinism**: run `pipeline.generate(img, seed=42)` 100 times and verify all outputs are bit-identical. Any non-determinism indicates an unguarded `torch.randn` call inside the loop that ignores the seed.

**Multi-control scale additivity**: with two identical control images at scale=0.5 each, the output should be approximately equal to a single control at scale=1.0. This validates that additive stacking is correctly proportional.

**Concurrent generation (100 requests)**: start the FastAPI server with `uvicorn src.server:app --port 8000 --workers 1` and fire 100 concurrent POST `/generate` requests using `httpx.AsyncClient`. Verify all 100 return HTTP 200 with valid base64 PNG data. Measure p50 and p99 latency — with the stub UNet on CPU, p99 should stay under 10 seconds per request.

### ControlNetClient: base64-encoded control images

`ControlNetClient.java` wraps the Python FastAPI server using Spring Framework 6.1's `RestClient`. It accepts raw image bytes, base64-encodes them, and sends them to `POST /generate`. The response is decoded from base64 back to PNG bytes.

```java
@Component
public class ControlNetClient {
    private final RestClient http;

    public ControlNetClient(@Value("${controlnet.base-url}") String baseUrl) {
        this.http = RestClient.builder().baseUrl(baseUrl).build();
    }

    public byte[] generate(byte[] controlImageBytes, String mode, float scale, int steps) {
        String encoded = Base64.getEncoder().encodeToString(controlImageBytes);
        Map<String, Object> body = Map.of(
            "control_image", encoded,
            "mode", mode,
            "scale", scale,
            "steps", steps,
            "seed", 42,
            "prompt_embedding", List.of()
        );
        Map<String, Object> response = http.post().uri("/generate")
            .body(body).retrieve().body(Map.class);
        return Base64.getDecoder().decode((String) response.get("generated_image"));
    }
}
```

The `generate()` method is under 20 lines. The complexity lives in the Python pipeline, not in the Java client. This is the correct split: Java services should treat ControlNet as an HTTP microservice, not as a library to embed.

### Why base64 for image transfer?

JSON does not support binary data. The two standard options are base64 encoding (serialize bytes as a string) or multipart/form-data (send binary as a file upload). We use base64 because:
- The request payload is self-contained in a single JSON object alongside other fields (mode, scale, steps)
- Spring's `RestClient` handles JSON serialization automatically without multipart configuration
- At 64x64 PNG, the image is under 5KB — base64 overhead (33%) is negligible

For large images (512x512+), multipart/form-data is more efficient. The Java client would switch to `MultipartBodyBuilder` from `RestClient`.

## Where to go deeper

The foundational paper is **"Adding Conditional Control to Text-to-Image Diffusion Models" (Zhang et al., 2023)** — the ControlNet paper. Section 3.3 on zero-initialization and Section 4 on training are worth reading carefully after completing this lab. The ablation study shows that zero-init reduces training time by 3.1x compared to random initialization on the same hardware.

**DDIM** is described in **"Denoising Diffusion Implicit Models" (Song et al., 2020)**. The key contribution is deriving a non-Markovian diffusion process that allows arbitrary step skipping while using the same noise schedule as DDPM. Section 4 has the deterministic sampler derivation.

The **diffusers ControlNetModel source** (`src/diffusers/models/controlnet.py`) shows how the real architecture differs from our stub: 12 encoder stages, zero-conv at each stage, cross-attention conditioning on text embeddings at every residual block. Reading it after implementing this lab will be immediately comprehensible.

**ControlNet++** (2024) extends the original with improved training objectives that make the control signal more precise. It introduces a consistency loss that penalizes cases where the control image reconstructed from the generated output does not match the input control image.

## Related labs and questions

The prerequisite labs build the foundations used here:
- **lora-finetune**: LoRA uses the same zero-initialization trick (B=0 ensures delta-W=0 at step 0). ControlNet's zero-conv achieves the same guarantee for an entire convolutional branch.
- **llm-inference**: the DDIM sampler is conceptually similar to autoregressive decoding — each step conditions on all previous state. The key difference is that diffusion operates globally over the full image at each step, not left-to-right over tokens.
