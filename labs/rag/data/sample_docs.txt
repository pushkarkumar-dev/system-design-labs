Write-ahead logging (WAL) is a standard method for ensuring data integrity in databases. Before any changes are made to the actual data, those changes are first written to a log file. If the system crashes, the database can recover by replaying the log from the last checkpoint. PostgreSQL, MySQL InnoDB, and RocksDB all use WAL as their primary durability mechanism.

Consistent hashing is a technique used in distributed systems to distribute data across nodes in a way that minimizes reorganization when nodes are added or removed. In a consistent hash ring, each node is assigned a position, and each key maps to the first node encountered clockwise. Virtual nodes allow fine-grained load balancing by giving each physical node multiple positions on the ring. Dynamo, Cassandra, and Riak all use consistent hashing for data partitioning.

The LSM-tree (Log-Structured Merge-tree) is a data structure optimized for write-heavy workloads. Writes go first to an in-memory buffer called the memtable. When the memtable is full, it is flushed to disk as an immutable SSTable. Reads must check the memtable and multiple SSTables, which is why bloom filters are essential for performance. Compaction merges SSTables to reduce read amplification. RocksDB and LevelDB are production implementations of LSM-trees.

Reciprocal Rank Fusion (RRF) is a simple and effective method for combining ranked lists from multiple retrieval systems. The score for a document is computed as the sum of 1/(k + rank_i) across all retrieval systems, where rank_i is the document's rank in system i and k is a constant (typically 60) that prevents top-ranked documents from completely dominating. RRF does not require score normalization because it only uses rank positions, making it robust to differences in score distributions across systems.

Raft is a consensus algorithm designed to be more understandable than Paxos. A Raft cluster elects a leader node responsible for managing a replicated log. All writes go through the leader, which appends entries to its log and replicates them to followers. An entry is committed once a majority of nodes have acknowledged it. Raft handles leader failure through elections: a follower that hasn't heard from the leader starts a new election by incrementing its term and requesting votes.
